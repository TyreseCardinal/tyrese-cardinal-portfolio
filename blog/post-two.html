Refining the Frontend and Global Styles

October 22nd, 2025 Commit: Refine frontend structure and apply global styling presets

After building the foundation for my portfolio, tonight’s focus was on tightening up the front end. I finalized the color palette, adjusted font scaling, and gave each section consistent spacing. Everything now feels more cohesive and intentional.

What I Updated

I refined every HTML file to ensure class names match the styling presets defined in style.css. I also polished the grid and flex layouts, added utility classes for consistent margins and paddings, and improved how the site scales on smaller screens.

What I Learned

Design consistency isn’t just aesthetics—it’s workflow clarity. Having standardized utility classes for layout makes future edits way easier and keeps my code cleaner. It’s basically early-stage scalability.

Challenges

One annoying issue was the blinking cursor appearing on static text. Turns out it wasn’t a CSS bug at all—it was caret browsing in the browser itself. After confirming it wasn’t a code issue, I laughed, disabled it, and moved on.

Takeaway

With the base design finalized, I’m finally happy with how the portfolio feels. Tomorrow, I’ll start experimenting with interactive polish—hover animations, maybe a light/dark mode toggle, and small UX touches that make it feel personal.
